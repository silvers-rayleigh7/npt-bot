# tropa-bot — голосовой научный ИИ-гид «Путеводитель по научной тропе»

Полная техническая выжимка проекта: архитектура, решения, нюансы, развёртывание.
Секреты (ключи, токены, пароли) НЕ хранятся в репозитории — только в `.env` на сервере.

---

## 1. Что это и зачем

Telegram-бот: пользователь задаёт научный вопрос **голосом или текстом** → получает
**голосовой ответ** мужским голосом в стиле русского научпопа (Перельман, Ливанов,
журнал «Квант»). Объясняет «как устроен мир», обязательно называя научный принцип
явления (рэлеевское рассеяние, параметрический резонанс и т.п.).

**Контекст:** прототип для А. В. Гасникова (ректор, научные тропы), демонстрация на
ПМЭФ. Философия — «Манифест Гасникова»: метод вместо энциклопедии, поперечное сечение
реальности, научный ракурс вместо банального пересказа.

**Назначение сейчас:** личное некоммерческое использование (владелец + его аккаунты).

## 2. Архитектура

```
Пользователь (Telegram, мультиаккаунт по /start)
   │ голос / текст
   ▼
claude-tg (Python, systemd, VPS Нидерланды)         ← готовый каркас (форк)
   │ голос → Groq Whisper STT (ru) → текст
   │ персистентный процесс claude -p
   │ роутинг ответа в активный chat_id (current_chat.txt)
   ▼ текст (stdin NDJSON)
claude -p (Claude Code, OAuth-подписка, модель opus)
   │ читает CLAUDE.md (личность) + ~/.claude/skills/scipop-answer
   │ RAG: Grep/Read по content/ (выверенная база)
   │ opus формирует ответ (адаптивно: новичок/знаток)
   │ Bash → tts.py
   ▼
Yandex SpeechKit v1 (голос filipp, speed 1.25) → oggopus
   ▼
MCP send_telegram_file    → 🔊 голосовое (без формул)
MCP send_telegram_message → 📝 текст в два уровня (🔹 любознательным / 🔸 продвинутым с формулами)
```

**Латентность:** ~20-40 с на ответ (главное — opus + RAG). Осознанный trade-off
качество↔скорость. Ускорение при нужде: `CLAUDE_TG_MODEL=sonnet` в `.env`.

## 5. Развёртывание на новом сервере (в т.ч. перенос Max → Pro)

```bash
git clone <этот-репозиторий> tropa-bot && cd tropa-bot
bash setup.sh                          # пакеты, node, claude-tg, deps, apply_patches
claude setup-token                     # на машине с браузером под нужным аккаунтом
cp .env.example .env && nano .env      # заполнить ключи; chmod 600 .env
cp -r skills/scipop-answer ~/.claude/skills/
# systemd unit (см. README) → systemctl enable --now claude-tg
```

OAuth-токен переносится строкой; на новом аккаунте — новый `claude setup-token`.
Сменить любой параметр (модель, голос, скорость, аккаунт) = правка `.env` + рестарт.

## 6. Патчи claude-tg (`patches/apply_patches.py`, идемпотентны)

claude-tg ставится с PyPI; наши правки накладываются скриптом:
1. **Статус «🎙 Обрабатываю…»** вместо «Thinking…» (stream.py).
2. **Только голос:** не стримить текст ответа в чат; tool calls скрыты (verbose=0);
   при неудаче озвучки — короткий сигнал, не текст-простыня.
3. **Multi-account:** allowlist + `/start` авто-регистрация (лимит `CLAUDE_TG_MAX_USERS`)
   + динамический роутинг ответа в активный chat_id (`data/current_chat.txt`).
4. **send_voice:** `.ogg` → голосовое (PyPI-версия слала всё документом — фикс).
5. **send_telegram_message:** новый MCP-инструмент — текстовое сообщение (формула) вдогонку
   к голосу. Голос всегда без формул; если в ответе есть формула — она уходит вторым,
   текстовым сообщением (Юникод, моноширинный блок). Логика — в `CLAUDE.md`.
6. **STT:** Groq `language=ru` + научный prompt (media.py).
7. **import json** в bot.py (allowlist) + **IS_SANDBOX** для skip-permissions на root.

## 7. Ключевые решения и почему (история)

- **OAuth-подписка, не API.** Решает боль Гасникова «бот сожрал кредиты»: оплата
  подпиской, не поштучно. Легально для личного использования владельца.
- **opus + обязательный RAG.** Максимум качества/достоверности; принят trade-off ~20-40 с.
- **Yandex SpeechKit вместо ElevenLabs.** ElevenLabs — англо-движок: кривые ударения,
  «тянет». Yandex нативно-русский: правильные ударения из коробки. Голос `filipp`
  (v1) — стабильный мужской; v3 (anton) давал нестабильную доставку и андрогинный тембр.
- **Адаптивность уровня (голос) + два уровня (текст).** Бот определяет по вопросу новичка/
  знатока и калибрует под него ГОЛОС (глубина, лексика, длина). ТЕКСТ при этом всегда даёт оба
  уровня — 🔹 для любознательных (без формул) и 🔸 для продвинутых (механизм + формула). Формулы
  живут только в текстовом Уровне 2; голос их не зачитывает.
- **Манера в духе С. Капицы.** Размеренный темп, пауза перед ключевым словом, спокойная
  уверенность, уважительный регистр — манера подачи, без выдачи себя за реальное лицо.
- **Обязательный научный термин.** Каждый ответ называет принцип (рэлеевское рассеяние и т.п.).
- **Multi-account через `/start`.** Владелец с нескольких своих аккаунтов (не третьи лица).

## 8. Решённые баги (чтобы не повторять)

- **VOICE_MESSAGES_FORBIDDEN** — приватность Telegram получателя блокирует голосовые;
  лечится в настройках аккаунта (Конфиденциальность → Голосовые → Все).
- **PyPI claude-tg слал .ogg документом** — добавлена `send_voice`-маршрутизация.
- **`json` не импортирован** в bot.py → `/start` падал → 2-й аккаунт не регистрировался.
- **Yandex v3 streaming** — нестабильная доставка + андрогинный голос → откат на v1.
- **`.pyc`-кэш** — после патчей чистить `__pycache__`, иначе старый байткод.

## 10. Эксплуатация

- Бот автономен (systemd `Restart=always`, переживает перезагрузки).
- Логи: `journalctl -u claude-tg -f`.
- Рестарт: `systemctl restart claude-tg`.
- Тесты: `python3 -m pytest tests/ -v`.
- Yandex грант покрывает ~10-12 тыс. ответов; остаток — Биллинг Yandex Cloud.
