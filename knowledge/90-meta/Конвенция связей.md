# Конвенция связей и структуры базы НПТ

Правила, по которым строится вся база. Их же проверяет авто-смотритель (хук).

## Полки (Johnny.Decimal)
`10 Сюжеты` · `20 Локации` · `30 Карты (MOC)` · `40 Проекты` · `90 Мета` · `assets` (медиа).

## Frontmatter сюжета (вариант B - типизированные связи)
```yaml
---
type: сюжет            # сюжет | локация | карта | проект | мета | home
code: WP011            # код из банка (если есть)
status: готов          # идея | черновик | готов | верифицирован
tags: [астрономия, геометрия]
cross_section: "общий принцип одной фразой"   # для ассоциативных связей
related: ["[[Солнечные часы]]"]                # связан КАК: по смыслу/принципу
source:  ["[[Учебник Ливанова]]"]              # откуда факты
used_in: ["[[Тропа Иннополис]]", "[[Чат-бот гид]]"]  # где используется
levels: [Кратко, Как это работает, Глубже]     # уровни А/Б/В
---
```

## Три типа связей
1. **Иерархическая** - MOC → сюжет (`[[Обсерватория]]` в карте темы).
2. **Ассоциативная** - сюжет ↔ сюжет по общему `cross_section`.
3. **Типизированная** - через frontmatter `related / source / used_in / part_of`.

## Железные правила
- **Нет сирот:** у каждой заметки минимум одна связь (в текст или во frontmatter).
- **Связывай сразу:** `[[...]]` ставится в момент упоминания, не «потом».
- **Каждый сюжет атомарен:** одна идея = один файл.
- **У каждого сюжета 3 уровня** (Кратко / Как это работает / Глубже).
- **Каждая локация и сюжет ведёт наверх** к своему MOC и к [[_home]].
