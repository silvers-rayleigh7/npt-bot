# Гард единого источника

Автоматическая защита правила «сюжеты — только в базе `knowledge/`». Правило само по себе
описано в `tools/README-edinyj-istochnik.md`; здесь — как оно **принуждается** машиной,
чтобы не полагаться на память.

## Зачем

21.07.2026 весь сеанс сюжеты правились прямо в `content/storylines/` (сайт), в обход базы —
и это заметили только на этапе коммита. Разбор: правки шли **из чужого воркдерева**, где
cwd не `~/Projects/tropa`, поэтому проектный хук в `tropa/.claude/` такое не поймал бы.
Вывод: защита обязана срабатывать **глобально и по пути редактируемого файла**, а не по cwd.

Работу тогда не потеряли — обратный синк `vault_sync_from_site.py` вернул тела в базу без
потери связей. Но это разовое спасение задним числом; гард закрывает причину.

## Что делает `tools/tropa-workflow-guard.sh`

Один скрипт, два режима:

| Режим | Когда | Действие |
|-------|-------|----------|
| `edit` | PreToolUse на Edit/Write | Если путь под `*/content/storylines/*.md` или `*/tg-bot/content/*.md` — **блокирует** правку (exit 2) с подсказкой: правь в базе, затем `gen_site_from_vault.py`. Правки шаблонов, кода, самой базы — пропускает. |
| `session` | SessionStart | Если сессия про «Тропу» (cwd или git-remote содержит `tropa`/`npt-bot`) — печатает правило и сверяет базу с сайтом (`gen_site_from_vault.py`, сухой прогон). Расхождение → предупреждение. В чужих проектах молчит. |

Блокируется только **Edit/Write** (ручная правка). Разнос `gen_site_from_vault.py` пишет через
python, поэтому гард ему не мешает — это и есть единственная разрешённая дверь к продукту.

## Подключение (глобальные хуки Claude Code)

Хуки стоят в `~/.claude/settings.json` — **глобально**, чтобы ловить правки «Тропы» из любого
терминала и любого cwd. Скрипт-обработчик — здесь, в базе (единый источник и для правил):

```json
{
  "hooks": {
    "SessionStart": [
      { "hooks": [{ "type": "command",
        "command": "bash ~/Projects/tropa-bot/tools/tropa-workflow-guard.sh session" }] }
    ],
    "PreToolUse": [
      { "matcher": "Edit|Write", "hooks": [{ "type": "command",
        "command": "bash ~/Projects/tropa-bot/tools/tropa-workflow-guard.sh edit" }] }
    ]
  }
}
```

## Правильный цикл (напоминание)

1. Правишь сюжет в базе `knowledge/10-syuzhety/<Название>.md`.
2. `python3 tools/gen_site_from_vault.py --apply --bot` — разнос на сайт и бота.
3. Сайт: `build.py` в репо Даниила → коммит → пуш → `tools/sync-site-mirror.sh` (зеркало).
4. Бот: `scp` контента на сервер.

Если сайт уже поправили напрямую — сначала `tools/vault_sync_from_site.py` (вернуть в базу),
и только потом продолжать из базы.
