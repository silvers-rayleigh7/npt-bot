# ✅ Верификация контента (bottleneck)

Главный bottleneck проекта - неавтоматизируемая ручная работа. Ведёт [[Константин Цыбиков]]. Контекст - демо вице-премьеру ([[Визит 17 июня]]), ошибки недопустимы.

## Проблема
- Claude совершает много ошибок. У [[Константин Цыбиков|Константина]] 5 разных аккаунтов в нейросетях: Claude и ChatGPT не видят ошибки друг друга.
- Процесс: найти реальную ошибку → прогнать через Claude, ChatGPT-5, Perplexity - не покрывают задачу полностью.
- Нужен «консенсус» моделей + ранжирование ошибок по важности (одна модель считает мелочь фатальной, другая - ненужной).
- Нет собеседника для рассуждения - начинаешь слишком доверять модели.
- Скил SkillFeedReviewer ранжирует приоритеты исправления.

## Action
- Автоматизировать верификацию (консенсус моделей, ранжирование ошибок) - [[Константин Цыбиков]] + первокурсники.
