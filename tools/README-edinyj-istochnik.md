# Единый источник: база знаний → сайт и ТГ-бот

Контент сюжетов ведём **в одном месте — базе `knowledge/`**. Оба продукта (сайт Даниила
и наш ТГ-бот) получают контент из неё. Правишь сюжет один раз в базе — обновляются оба.

## Мост база ↔ сайт
Каждая заметка сюжета, опубликованного на сайте, имеет во frontmatter поле **`site_slug`** —
имя файла на сайте (`content/storylines/<site_slug>.md`). Оно связывает заметку с сайтом.
Заметки без `site_slug` — контент только базы (черновики/доп. станции), на сайт не идут.

Заметка базы содержит:
- **сайтовые поля** (для генерации): `code`, `region`, `tags`, `geo`, `routes`, `site_slug`;
- **knowledge-поля** (только база): `cross_section`, `related`, `source`, `used_in`, `type`, `status`;
- **тело** — дословно как на сайте (со схемами `/assets/*.svg`);
- раздел **`## Связи`** — только база, при генерации сайта отбрасывается.

## Скрипты

| Скрипт | Что делает |
|--------|------------|
| `tools/new_storyline.py "Название" <slug> [код]` | создаёт заготовку сюжета в базе по канону (frontmatter + 3 уровня + «## Связи» + чек-лист) |
| `tools/gen_site_from_vault.py` | **разносит базу → сайт + бот**: добавление / изменение / **удаление** сюжетов, синк схем `.typ`. Без флагов — сухой прогон; `--apply` (сайт), `--bot` (ТГ-бот), `--check` (строгая сверка) |
| `tools/vault_sync_from_site.py` | обратный синк сайт → база (если кто-то правил прямо на сайте), без потерь |

## ⛔ Единственный путь: всё через базу
Сюжеты НЕ создаём, НЕ правим и НЕ удаляем напрямую в `content/storylines/` (сайт) или
`tg-bot/content/storylines/` (бот). **Только в базе `knowledge/`**, затем разносим скриптом.
`gen_site_from_vault.py` — единственная дверь: он и добавит новые, и обновит изменённые,
и удалит те, что убрали из базы. Прямая правка продукта затрётся при следующем разносе.

## Полный цикл сюжета (добавить / изменить / удалить)
1. **Создать:** `python3 tools/new_storyline.py "Название" <slug> <код>` → пишем текст, сноски-источники,
   формулы, схему `content/figures/<slug>-1.typ` (кладём и в `knowledge/assets/figures/`), добавляем в MOC.
   **Изменить** — правим заметку в базе. **Удалить** — удаляем заметку из базы.
2. **Проверить базу:** `python3 ~/.claude/skills/obsidian-keeper/keeper_check.py knowledge` (0 ERROR).
3. **Сухой прогон:** `python3 tools/gen_site_from_vault.py --bot` — видим, что добавится/изменится/удалится.
4. **Разнести:** `python3 tools/gen_site_from_vault.py --apply --bot`.
5. **Сайт:** собрать `PATH="$PWD/.tools:$PATH" python3 build.py` (Typst 0.13.1) в репо Даниила →
   закоммитить **только файлы нового/изменённого сюжета** (существующий PDF-шум откатить, см. DEV-NOTES) → пуш.
6. **Бот:** контент в репо обновлён; на живом боте появится после деплоя (scp из `tg-bot/` на сервер).

Сайт и ТГ-бот едят **один формат** сюжета, поэтому один генератор питает оба.

## Рабочий цикл (обновить сюжет)
1. Правим сюжет **в базе** (`knowledge/10-syuzhety/<Название>.md`).
2. `python3 tools/gen_site_from_vault.py` — сухой прогон, смотрим, что меняется (и что не ломается).
3. `python3 tools/gen_site_from_vault.py --apply` — пишем в `~/Projects/tropa/content/storylines/`.
4. Собираем сайт (`PATH="$PWD/.tools:$PATH" python3 build.py` в репо Даниила) и пушим в `MerkulovDaniil/tropa`.
5. Даниил не участвует — весь цикл на нашей стороне.

## Гарантия «ничего не ломается»
`gen_site_from_vault.py` перед записью **сверяет** сгенерированное с текущим сайтом:
значения frontmatter + тело + побайтово. Проверено 06.07.2026 — **40/40 сюжетов байт-в-байт**.
Если появится расхождение — скрипт остановится с ошибкой и НЕ применит изменения.
