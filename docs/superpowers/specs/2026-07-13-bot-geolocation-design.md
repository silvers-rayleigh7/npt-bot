# Дизайн: геолокация в ТГ-боте «Тропа»

Дата: 2026-07-13 · Статус: одобрен (дизайн), готов к плану реализации

## Цель
Пользователь отправляет боту геолокацию → бот показывает ближайшие научные точки (сюжеты
с координатами) и по выбору рассказывает про точку (голос + текст, как обычно), в том числе
обсуждает тропу и что вокруг. Фаза 1 — разовая отправка; Live Location — задел на фазу 2.

## Контекст (что уже есть)
- Транспорт — пакет **`claude_tg`** (uv tool на сервере `/root/.local/.../claude_tg/`):
  - `bot.py` — Telegram `Application` + хендлеры (`CommandHandler`, `MessageHandler`),
    пишет `data/current_chat.txt`, буферит фото. **Локацию сейчас НЕ обрабатывает.**
  - `runner.py` — персистентный процесс `claude -p`, метод `inject(prompt)` / `run(prompt)`.
  - `mcp_server.py` — MCP-инструменты ответа: `send_telegram_message`, `send_telegram_file`
    (голос oggopus), `get_conversation_context`.
- Контент бота — `content/storylines/*.md` (генерятся из базы, у 13 иннополисских + ряда
  региональных есть frontmatter `geo: [lat, lon]`). Всего с `geo` ~37 сюжетов.
- `CLAUDE.md` бота — есть раздел-заглушка «Геоконтекст» (soft, без матчинга).
- В репо бота есть **`patches/apply_patches.py`** — штатный механизм патчить форк транспорта.

## Архитектура (разделение ответственности)
- **Транспорт (`bot.py`, детерминированно):** ловит сообщение-локацию, считает ближайшие точки
  по **Haversine**, формирует структурированную гео-подсказку, `inject` в `claude -p`.
- **`claude -p` (нарратив):** получив гео-подсказку, показывает список 2–3 ближайших и по выбору
  рассказывает — через существующие MCP-инструменты (голос + текст). Расстояния не считает.
- **Единый источник:** координаты берём из `content/storylines/*.md` (те же файлы, что бот и так
  читает через RAG). Добавили сюжет с `geo` в базе → после генерации бот его видит. Отдельный
  индекс не нужен.

## Компоненты
1. **`geo_match(lat, lon)`** — новый хелпер (в патче транспорта или отдельный модуль):
   парсит `geo` из `content/storylines/*.md`, считает Haversine до каждой точки, возвращает
   топ-3 ближайших `[(title, slug, distance_m), …]`.
2. **`handle_location(update, context)`** — новый `MessageHandler(filters.LOCATION, …)` в `bot.py`:
   - берёт `update.message.location` (фаза 1) — `latitude`, `longitude`;
   - вызывает `geo_match`;
   - формирует гео-подсказку и `inject` в runner (как текстовый хендлер отдаёт промпт claude).
3. **Кнопка запроса локации** — `ReplyKeyboardMarkup([[KeyboardButton("📍 Отправить геолокацию",
   request_location=True)]])`, показывается на `/start` и по команде `/where` (или в подсказке).
4. **Гео-подсказка для claude** (текст inject), например:
   ```
   [ГЕО] Пользователь прислал геолокацию (55.7514, 48.7312).
   Ближайшие научные точки: 1) Белая берёза — 120 м; 2) Обсерватория — 300 м;
   3) Маятниковая роща — 500 м. Покажи их списком (кнопки/цифры), дай выбрать,
   затем расскажи про выбранную как обычно (голос + текст).
   ```
5. **`CLAUDE.md` бота** — раздел «Геоконтекст» переписать: как обрабатывать `[ГЕО]`-блок
   (показать список, дождаться выбора, рассказать; если выбор — номер, взять соответствующий сюжет).

## UX-поток (фаза 1 — разовая)
```
/start (или /where) → кнопка «📍 Отправить геолокацию»
  → юзер жмёт → Telegram шлёт message.location
  → транспорт: топ-3 ближайших → inject [ГЕО] claude
  → бот: «Рядом с тобой: 1. Белая берёза — 120 м; 2. Обсерватория — 300 м;
     3. Маятниковая роща — 500 м. Про что рассказать?»
  → юзер: «1» / тап → бот рассказывает (голос + текст) + можно обсудить
```

## Правила и краевые случаи
- **Радиус fallback:** если ближайшая точка дальше **3 км** — бот честно говорит «рядом точек нет,
  ближайшая — X в N км» и предлагает рассказать про неё либо перейти к любой теме (не тупик).
- **Scope:** матчим ВСЕ сюжеты с `geo` (~37), не только маршрут Иннополиса.
- **Формат:** всегда топ-3 (или меньше, если сюжетов с geo мало), с расстоянием (м / км).
- **Единицы:** <1000 м → метры, иначе км с одним знаком.

## Фаза 2 — Live Location (задел, не реализуем сейчас)
- Тот же `geo_match`. Транспорт дополнительно ловит `edited_message.location` (обновления
  трансляции). Анти-спам: хранить per-user последнюю озвученную точку, срабатывать только при
  входе в радиус ~150 м и не повторять ту же точку. Спроектировано так, что подключается сверху.

  новый модуль geo-матчинга), затем перезапуск сервиса `claude-tg`. `CLAUDE.md` — scp как контент.
- Сервер `144.124.250.77`, сервис systemd `claude-tg`, рабочая папка `/root/tropa-bot`.

## Критерий приёмки (outcome-level)
- Реальный проход в Telegram: нажал кнопку → пришёл список ближайших с расстояниями → выбрал →
  бот рассказал голосом+текстом. Проверка расстояний на известных координатах (Haversine sanity).
- Fallback при «далеко» отрабатывает (не падает, даёт осмысленный ответ).

## Явно НЕ входит (YAGNI)
- Live Location (фаза 2), карта-виджет в боте, навигация/маршрутизация между точками, МАКС.
