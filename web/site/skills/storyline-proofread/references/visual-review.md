# Визуальная вычитка: как смотреть рендер

Чтение исходника НЕ заменяет просмотр рендера. Визуальные критики обязаны
открыть и HTML каждого таба, и PDF каждого уровня.

## Поднять сайт и снять скриншоты HTML

```bash
cd /root/tropa && python3 build.py
cd site && (nohup python3 -m http.server 8777 >/tmp/t.log 2>&1 &) ; sleep 1
```

Скриншотер (`/tmp/shot.py` — patchright+chromium, device_scale_factor=2):

```python
from patchright.sync_api import sync_playwright
import sys
url, out, full = sys.argv[1], sys.argv[2], len(sys.argv) > 3
with sync_playwright() as p:
    b = p.chromium.launch()
    pg = b.new_page(viewport={"width":1280,"height":900}, device_scale_factor=2)
    pg.goto(url, wait_until="networkidle"); pg.wait_for_timeout(800)
    pg.screenshot(path=out, full_page=full); b.close()
```

```bash
python3 /tmp/shot.py "http://localhost:8777/storylines/<slug>/index.html" /tmp/v.png full
```

- Снимать КАЖДЫЙ таб (переключить и переснять) и мобильную ширину (viewport 390).
- Затем **прочитать PNG** (Read tool) и придираться по чек-листу критика 5.

## Открыть и проверить PDF уровней

PDF лежат в `site/storylines/<slug>/{a,b,c}.pdf`. Открыть их (Read tool читает
PDF постранично) и придираться по чек-листу критика 6 — книжная планка.

```bash
ls /root/tropa/site/storylines/<slug>/*.pdf
```

PDF собирает `build_pdf` (Typst, Libertinus Serif, буквица droplet, footer
`assets/glyphs/divider.svg`). Шрифт страницы и формул — Libertinus Serif:
любая добавляемая иллюстрация должна использовать тот же серифный шрифт.

## Книжная планка (что считается «топ»)

Сравнивать не с «прилично для веб-странички», а с разворотом дорогой
научно-популярной книги:
- воздух и ритм: поля, межстрочье, расстояния между блоками выверены;
- ни одной висячей строки/предлога, ни одного кривого переноса;
- иллюстрации — часть полосы, а не «вставленная картинка»: единый штриховой
  стиль, серифные подписи, выровнены по сетке;
- формулы выглядят набранными (как в LaTeX/Libertinus), а не «напечатанными».

Любое отклонение от этой планки — находка визуального критика, а не «и так
сойдёт».

## Визуальные дефекты — чек-лист (2026-06-02, из правок Даниила)

Критик визуала ОБЯЗАН проверять по каждой схеме и каждой странице:

1. **Текст не залезает на рисунок.** Подписи внутри CeTZ-схемы не должны
   пересекать линии/дуги/фигуры. Типичная ловушка — веер дуг (волны, лучи)
   заходит в дорожку подписи. Фикс: вынести подпись в чистую полосу между
   элементами и/или сократить (стрелка уже несёт направление — слова дублируют).
   Инцидент: `uslysh` — «эхо вернулось, ещё s» под дугами звуковых фронтов.
2. **Квадратные/высокие схемы (w/h < 1.3) — ограничивать по высоте**, иначе
   съедают всю полосу. В PDF `build.py` (`_fig_size_directive`) ставит
   `height: 7.6cm`; в HTML `.fig img{max-height:340px}`. Проверять, что новые
   высокие фигуры не ломают вёрстку страницы.
3. **Первый уровень (Кратко) открывается визуалом, а не стеной текста** —
   «книга на первый взгляд». Если у L1 есть СВОЙ образ (не дубль L2) — дать
   открывающую схему (идиом принятых: колонка geo-srez, арка, шкала). НО знать
   меру: где хук сильнее голым текстом — не лепить схему ради галочки.
