# Контракт формата сюжета

Единый источник сюжета — `content/storylines/<slug>.md`. `build.py` разворачивает
его в страницу сайта (табы-уровни) и в PDF каждого уровня. Формат — это **контракт
с билдом**: нарушишь — рендер сломается тихо (метка покажется буквально, дек
прилипнет к тексту, появится лишний таб).

## Frontmatter

```yaml
---
title: <Название>            # обязателен
code: WP###                  # код точки, опц.
tags: [тег, тег]             # 1–3 тега
geo: [lat, lon]              # есть → точка на карте/маршруте; нет → сюжет без точки
routes: [innopolis]          # членство в маршрутах
---
```

## Тело: ровно три раздела-уровня

```
## Кратко               # L1 — голос у точки, 80–150 слов, без формул
## Как это работает      # L2 — механизм + ДИСПЛЕЙ-формула + численная оценка
## Глубже                # L3 — углубление, источники, связки
```

**Визуал и формула — обязательны (не опция):** каждый сюжет обязан иметь
≥1 векторную схему, а L2 — отцентрированную дисплей-формулу (`$$…$$`). Стена
текста без видимой схемы/формулы = blocker (см. `critic-panel.md`, раздел
«Иллюстрации как часть выхода»).

**Артефакт на КАЖДОМ уровне (Кратко/Как это работает/Глубже):** каждый таб
обязан нести ≥1 артефакт помимо текста. Допустимые типы и их рендер:
- **схема** — `![подпись](/assets/figures/<slug>-N.svg)`, источник `.typ` (CeTZ,
  стиль — `figure-style.md`). Квадратные/высокие (`w/h<1.3`) ограничиваются по
  высоте автоматически (`_fig_size_directive`).
- **дисплей-формула** — `$$…$$` на своей строке (через mitex; обязательна в L2).
- **таблица реальных данных** — markdown-таблица `| … | … |` с строкой-
  разделителем `|---|---|`. Рендерится в PDF книжным booktabs-стилем
  (`_md_table_to_typst`: верхняя/средняя/нижняя линейки, жирная шапка) и в HTML
  нативно (markdown `extra`). ИДЕАЛЬНО для уровня Глубже: скорости, методы,
  системы, размерности, константы. Числа — только из текста/проверенных
  источников (no_unfounded_claims), со сноской `[^id]`.
- **график** — это разновидность схемы (оси + кривая/точки в CeTZ).

Уровень Глубже чаще всего просит **таблицу** (компактная сводка реальных данных)
или **график**; методологический уровень — **схему-процесс**.

## Обращение к читателю — единое «ты» (blocker)

Голос тропы — научный гид, обращается **на «ты»** (`bot/config/style.md`:
«Ты — научный гид… посмотри сейчас на…»). Эталон — `belaya-bereza` («Выйди
в жару…», «Приложи ладонь…»). Любой сдвиг на «вы» (включая императивы
`посмотрите/найдите/представьте` и местоимения `вы/вам/вас/ваш`) = blocker.
Исключение одно: картографический идиом «вы здесь» на пине карты.

**Как проверять (НЕ частотным счётом!):** канон задаёт `style.md`, а не
большинство голосов в файлах — агенты массово дрейфуют в «вы», и счёт «вы vs ты»
покажет неверного победителя. Сверяйся со `style.md` + эталоном, потом приводи к «ты».
Скан остатков (любые 2-мн формы — труднее, чем кажется: `-ете/-ёте/-ите/-йте/-ьте`):

```bash
cd content/storylines
grep -rnEoi '\b[а-яё]+(ете|ёте|ите|йте|ьте)\b|\b(вы|вам|вас|вами|ваш[а-яё]*)\b' *.md \
  | grep -v 'вы здесь' | grep -viE 'орбите|зените|счёте|учёте|планете|пакете|расчёте'
# остаток = только существительные на -ите/-ете + «вы здесь». Глаголы 2-мн → blocker.
```
Инцидент 2026-06-02: частотный скан с дырявым регэкспом → ошибочно унифицировал
в «вы» (неверная сторона) + неполная конверсия дала смесь «ты/вы» в одном
предложении. Урок: читай канон, скань ВСЕ окончания, проверяй грамматику глазами.

- Заголовки уровней — **точно** `Кратко` / `Как это работает` / `Глубже`
  (билд маппит их в роли a/b/c). Допустимо иметь не все три (например, только
  Кратко + Глубже).
- **Никаких других `##`** внутри сюжета. Чужой `##` билд примет за новый
  уровень → лишний кривой таб. (Инцидент: `## Голос в фокусе`, `## **Брёвна…**`.)
- **Никаких `###`.** Подзаголовок — это одиночная жирная строка (см. ниже),
  не `### **X**`: на первом блоке `### **X**` даёт дек со звёздочками.

## Дек и подзаголовки

`build.py` (`_split_deck`, `md_levels`) трактует структуру так:

- **Дек** — ПЕРВАЯ одиночная жирная строка уровня. Рендерится курсивным
  подзаголовком под названием (на странице) и под масштхедом (в PDF). Коротко,
  < 80 символов. Обычно — вопрос-хук («Где здесь миллион лет?»).
- **Подзаголовок** — любая последующая одиночная жирная строка → `<h3>`.
- **Одиночная жирная строка** = вся строка есть `**…**`, после закрывающих
  `**` ничего нет.

**КРИТИЧНО: после дек-строки и каждого подзаголовка — пустая строка.**
`**Заголовок**` сразу со следующей строкой текста (без пустой) → билд склеит
их в один блок → дек НЕ извлечётся, жирное прилипнет инлайном к абзацу.

```
**Где здесь миллион лет?**          ← дек

Цветные полосы на этой колонке…     ← абзац (через пустую строку!)

**Что наблюдаем**                   ← подзаголовок

Посмотри на колонку снизу вверх…
```

- **Инлайн-акцент** — `**Слово.** дальше текст…` — это НЕ заголовок (после
  `**` идёт текст). Билд оставляет как `<strong>` внутри абзаца. Трогать не надо.

## Формулы и математика

- **Любая математика в тексте — в `$…$`**, не обычными символами. Переменные и
  выражения в прозе (`α`, `(1−α)·P`, `T=2π√(L/g)`) обязаны быть формулами:
  пиши `$\alpha$`, `$(1-\alpha)\cdot P$`, `$T = 2\pi\sqrt{L/g}$`. Синтаксис —
  LaTeX (KaTeX рендерит на вебе; билд конвертирует в Typst-math для PDF через
  `_clean_math`/`_latex_to_typst`). Голый `α` или `·` в тексте — находка
  научного критика.
- **Ключевая формула метода в L2 — ОБЯЗАТЕЛЬНО display**, на своей строке:
  `$$ d \approx \sqrt{2Rh} $$`. Метод, спрятанный только в мелкой инлайн-`$…$`
  посреди абзаца, на рендере не виден → blocker (см. `critic-panel.md`).
  Инлайн-`$…$` — для переменных/мелких выражений в прозе, не для главной формулы.
  Для TTS дублировать смысл словами в соседнем абзаце.
- Не оставлять «…задаётся формулой:» с пустотой после — формула должна быть.

## Запрещённые строительные метки (отбраковка структурного критика)

Это артефакты генератора. В целевом виде их быть не должно — заменить на
чистый дек/подзаголовок или убрать:

- `**Заголовок**`, `**Заголовок QR**` → сделать деком сам заголовок уровня.
- `**Хук**`, `**Введение**` → убрать метку, абзац остаётся телом.
- `**Уровень A/Б/В …**`, `**Блок N. …**`, `**N. …**`, `### **N. …**` →
  оставить только осмысленную часть как подзаголовок.
- `**N.N Стенд …**`, `**Нижний блок …**`, `**Комментарий**`, `⚠ …`,
  CAPS-пометки («НУЖНЫ ТОЧНЫЕ КООРДИНАТЫ…»), «(опционально/на согласовании)» —
  внутренние производственные пометки. Убрать из публички; о существенном
  удалении — сообщить владельцу.

## Контрольный grep (структурный критик)

```bash
cd content/storylines
# (1) строительные метки — фиксированные строки/паттерны (без класса [А-Я]{n}: он
#     в UTF-8-локали матчит байты кириллицы и даёт ЛОЖНЫЕ срабатывания на каждой строке)
grep -rnE '^### |^## \*|Заголовок|Уровень [АВСABCБ]|Блок [0-9]|\*\*Хук|\*\*Введение|Нижний блок|\*\*Комментарий|⚠|Стенд \(информац' *.md
# (2) CAPS-пометки (≥5 заглавных подряд) — корректно через PCRE-класс \p{Lu}, с UTF-8 локалью
LC_ALL=C.UTF-8 grep -rnP '\p{Lu}{5,}' *.md
# чисто = обе команды пусты (exit 1). Плюс: у каждого файла только канонические ## (Кратко/Как это работает/Глубже).
```
