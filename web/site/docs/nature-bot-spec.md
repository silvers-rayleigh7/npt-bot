# ТЗ: Telegram-бот «Природный гид»

> Бот отвечает на вопросы о природе на тропе — голосом или текстом — в каноне
> русского научпопа. Опирается на готовые вычитанные сюжеты, но умеет уходить
> в сторону: на любой вопрос строит ответ тем же методом, что и эталонные сюжеты.
>
> Визуальная схема и сценарии: **/bot/** на tropa.fmin.xyz.

## 1. Что делает бот (одной фразой)

Посетитель спрашивает «почему здесь обрывистый берег?» голосом — бот отвечает
голосом коротким сюжетом уровня А, предлагает «копнуть глубже» (Б → В) и, если
готового сюжета нет, сам сочиняет ответ в том же стиле и с той же глубиной.

## 2. Движок

| Слой | Технология | Роль |
|------|-----------|------|
| Канал | **claude-tg** | Telegram-шлюз: приём текста/голоса/геолокации, отправка voice + текст |
| Мозг | **Claude Code** (агент) | Понимание вопроса, retrieval, композиция ответа по канону |
| Стиль/глубина | скилл **`outdoor-scipop-architect`** | Уровни А/Б/В, 8 фильтров манифеста, метод сечения, канон Перельмана/Ливанова |
| Знания | **RAG** | Вычитанные сюжеты + экспонаты + материалы бота (см. §4) |
| Голос → текст | **Groq Whisper** (STT) | Распознавание голосового вопроса |
| Текст → голос | **ElevenLabs** (TTS) | Озвучка ответа голосом учёного |

Принцип: **claude-tg = руки и уши, Claude Code = голова, скилл = манеры, RAG = память.**

## 3. Поток обработки одного вопроса

1. Пользователь шлёт **голос / текст / live-локацию** в Telegram.
2. claude-tg принимает апдейт. Голос → **Groq Whisper** → текст.
3. Claude Code получает: текст вопроса + (опц.) координаты + текущий уровень глубины пользователя.
4. **Retrieval** по базе знаний (§4): находит наиболее релевантный сюжет/экспонат.
5. **Композиция ответа** под управлением скилла:
   - есть подходящий сюжет → отдать нужный **уровень** (А по умолчанию);
   - сюжета нет → построить ответ **на лету** по методу сечения и канону стиля (см. §6).
6. Текст ответа → **ElevenLabs** → голосовое сообщение. Отправляются **и голос, и текст**.
7. Бот предлагает следующий шаг: «копнуть глубже» / «другой сюжет рядом».

## 4. База знаний для RAG (всё уже в репозитории)

| Источник | Путь | Что даёт |
|----------|------|----------|
| Вычитанные сюжеты | `content/qr/storylines/ST-*.md` + `content/storylines.vetted.yaml` | 40 эталонных сюжетов, тексты уровней А/Б/В |
| Экспонаты тропы | `content/exhibits/*.md` + `content/exhibits.yaml` | 41 стенд (привязка сюжета к точке) |
| Материалы бота | `BRIEF.md`, `content/sources/`, `bot/config/style.md` | Tone of voice, исходные материалы |
| Канон стиля | `skills/outdoor-scipop-architect/references/` | Фильтры, метод сечения, playbook |

Индексация: чанки по уровням (А/Б/В — отдельные чанки), метаданные `{id, section, topic, level, exhibit, tags}`. При retrieval сначала ищем по сюжету, потом отдаём чанк нужного уровня.

## 5. Адаптивные уровни ответа

Уровни берём из скилла (`references/levels-a-b-c.md`):

- **А — «почувствуй»**: 80–150 слов, хук + действие + одна мысль. Без формул. *По умолчанию.*
- **Б — «пойми, как работает»**: 200–400 слов, принцип одной фразой + сечение + одна формула.
- **В — «глубже»**: 600–1500 слов, полное сечение, численная глубина, метод науки, открытый вопрос.

Управление уровнем:
- команды `/глубже`, `/проще`, кнопки **А · Б · В** под ответом;
- авто-подстройка под формулировку («расскажи формулами» → Б/В; «в двух словах» → А);
- бот **помнит выбранный уровень** в рамках диалога.

## 6. Готовые сюжеты vs отклонение в сторону

- **Якорь — 40 вычитанных сюжетов.** Если вопрос попадает в один из них — отвечаем его выверенным текстом (надёжно, без галлюцинаций).
- **Свобода — любой вопрос.** Нет готового сюжета → агент строит ответ **тем же методом**:
  1. поперечное сечение (один принцип → ≥3 явления из разных доменов с механизмом);
  2. структура уровня (хук → действие/наблюдение → мысль → якорная связь);
  3. 8 приёмов канона (бытовой заход, парадокс, числовая конкретика, формула как кульминация, открытый вопрос…);
  4. привязка к месту, если есть координаты.
- **Граница честности:** на лету бот не выдумывает числа/факты как достоверные — отделяет проверенное от прикидки (фильтр 1 манифеста).

## 7. Команды и режимы

| Команда | Действие |
|---------|----------|
| голос / текст | задать вопрос |
| `/гид` + live-локация | проактивный режим: рассказ у ближайшей точки |
| `/глубже`, `/проще` | сменить уровень (А ↔ Б ↔ В) |
| `/рядом` | какие сюжеты есть поблизости |
| кнопки А·Б·В | прямой выбор уровня |

## 8. Нефункциональные требования

- **Офлайн-деградация:** живой диалог требует сети; pre-generated mp3 эталонных сюжетов работают офлайн.
- **Голос:** один постоянный «голос учёного» (ElevenLabs), кэш озвучек по хешу текста.
- **Стоимость:** Groq STT — free tier; кэш TTS и готовых сюжетов снижает вызовы ElevenLabs.
- **Антигаллюцинация:** ответы вне готовых сюжетов проходят чек-лист скилла (метод, сечение, честность) до отправки.
- **Расширяемость:** новый сюжет = добавить `ST-NN.md` + строку в `storylines.vetted.yaml`; RAG переиндексируется.

## 9. Этапы

1. **MVP:** текст-вопрос → RAG по 40 сюжетам → ответ уровня А/Б/В текстом. Команды уровней.
2. **Голос:** Groq STT на входе + ElevenLabs на выходе.
3. **Гид:** live-локация → ближайшая точка → проактивный рассказ.
4. **Свободные ответы:** генерация вне готовых сюжетов по канону скилла + чек-лист.
